又是perf

Mar 5, 2017 | 工作相关 | Hits

最近又是半年一次的perf时期，然后又赶上组里的一个小hackathon，写的文档也差不多了，TL也要修产假去了，这一周似乎除了好好写写perf没有太麻烦的事情。

记得半年前第一次写perf有些不习惯，不知道怎样把普通的事写得厉害一些。现在似乎能好一点，也可以不吝于夸自己。但之前两个季度，也没做一个完整的project，因此也没什么特别突出好写的。

LG叫我直接要求promote，但我还是觉得自己表现不足，回头看看没什么拿得出手的project。然后他就灌输我一些想法，比如，很多人都觉得自己比较弱，但其实是很多中国人的通病，等等。之前有一次跟他讨论，有点脸红脖子粗的样子。但至少感觉在他的影响下有所进步，所以打算perf review的时候就跟manager提想要下次promote，让他看着给活干吧。说起自己的manager，有时候觉得不是太靠谱。一个是他是第一次当manager，他对技术很懂，但在对人的管理上似乎不是太成熟。首先，他经常skip我的one-to-one，有时候主动找他，他又忘了set up schedule。可能主要还是我做的事情不够紧要，让他不够关注。这个时候似乎就要自己更加主动了。另外，他会在会上讨论一些action item，但似乎并没有落实，下一次可能还是讨论老问题。

看到一个promote经验分享的note，并有请教另一个manager，感觉以下几点可以进行尝试：

  • 提前一年准备，跟manager说要promote（如果manager还是不靠谱考虑转组）
  • 多写文档
  • 完成quartly的project
  • 了解L4的要求
  • skip level的one-to-one，可以是monthly的，会前准备好
  • 平时多积累这段时间做了什么的note
  • 提前写perf

其实还是一场比较艰难的行为啊。真的很想最晚明年这个时候promote。
继续努力。
